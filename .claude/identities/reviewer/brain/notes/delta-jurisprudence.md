Spec deltas are for spec text that CONTRADICTS implemented reality (an ordering claim, a behavioral statement). Untyped prose being filled by a reasonable implementation choice needs code/README documentation, not a delta. Before demanding (or writing) a delta, read the ACTUAL spec text and check both precedent classes — the applied deltas (real corrections) and the undelta'd untyped-gap fills. A delta demanded on a misremembered prose claim is reflex process overhead.
